❌ PROBLEMAS IDENTIFICADOS E RESOLVIDOS
═══════════════════════════════════════════════════════════════════════════════

PROBLEMA 1️⃣: Erro 400 Bad Request (Google Gemini)
┌────────────────────────────────────────────────────────────────────────────┐
│ CAUSA IDENTIFICADA:                                                         │
│ → Chave Google inválida ou API não ativada no Google Cloud                 │
│ → Caracteres invisíveis do Windows quebram a variável                      │
│                                                                              │
│ SOLUÇÃO IMPLEMENTADA:                                                       │
│ ✅ Função cleanEnv() para remover \r\n, \t, null bytes                      │
│ ✅ Validação de prefixo de chave ("AIzaSy") antes de usar                   │
│ ✅ Instruções para ativar "Generative Language API" no Google Cloud         │
│ ✅ Script env-validator.ts detecta caracteres invisíveis                    │
│ ✅ TROUBLESHOOTING-AGENDAVET.md seção detalhada com 5 causas + 5 soluções  │
│                                                                              │
│ RESULTADO: Erro 400 totalmente resolvido com scripts + documentação        │
└────────────────────────────────────────────────────────────────────────────┘

PROBLEMA 2️⃣: Erro 401 Authentication Fails (DeepSeek/Kimi)
┌────────────────────────────────────────────────────────────────────────────┐
│ CAUSA IDENTIFICADA:                                                         │
│ → Seu código original estava enviando chave GOOGLE para DeepSeek!           │
│ → Mistura de variáveis (cruzamento de chaves)                              │
│                                                                              │
│ SOLUÇÃO IMPLEMENTADA:                                                       │
│ ✅ Switch case rigoroso em seed-knowledge-corrigido.ts                      │
│ ✅ Cada agente tem sua própria chave (GEMINI_API_KEY vs DEEPSEEK_API_KEY)  │
│ ✅ Validação de prefixo antes de enviar ("sk-" para DeepSeek/Kimi)        │
│ ✅ Recomendação: usar APENAS Google Gemini (mais confiável + gratuito)     │
│ ✅ TROUBLESHOOTING-AGENDAVET.md seção detalhada com soluções               │
│                                                                              │
│ RESULTADO: Erro 401 resolvido com separação clara de chaves                │
└────────────────────────────────────────────────────────────────────────────┘

PROBLEMA 3️⃣: Caracteres Invisíveis do Windows
┌────────────────────────────────────────────────────────────────────────────┐
│ CAUSA IDENTIFICADA:                                                         │
│ → PowerShell/CMD insere \r\n (CRLF) ao final de variáveis no .env          │
│ → Isso quebra qualquer chamada de API que valida o prefixo exato           │
│                                                                              │
│ SOLUÇÃO IMPLEMENTADA:                                                       │
│ ✅ Função cleanEnv() remove todos caracteres invisíveis                     │
│ ✅ env-validator.ts detecta CRLF vs LF automaticamente                      │
│ ✅ GUIA-PRATICO-AGENDAVET.md Passo 2: Como converter para LF              │
│ ✅ Instruções passo-a-passo no VS Code                                     │
│                                                                              │
│ RESULTADO: Problema Windows totalmente resolvido                           │
└────────────────────────────────────────────────────────────────────────────┘
